## API Examples

### Basic Usage
```cpp
Config config;
config.loadFromFile("game.json");

// Type-safe getters with defaults
int width = config.get<int>("graphics.resolution.width", 1920);
bool vsync = config.get<bool>("graphics.vsync", true);
float fov = config.get<float>("graphics.fov", 90.0f);

// Setting values
config.set("graphics.resolution.width", 2560);
config.set("graphics.vsync", false);

// Save changes
config.saveToFile();
```

### Hot-Reload
```cpp
config.enableHotReload([](const Config& cfg) {
    std::cout << "Config reloaded!" << std::endl;
    // Update engine settings
});

// In game loop
while (running) {
    config.checkForReload();
    // ... rest of loop
}
```

### Optional Values
```cpp
auto value = config.get<int>("optional.key");
if (value.has_value()) {
    useValue(value.value());
} else {
    useDefault();
}
```

## Performance Characteristics

- **Loading**: O(n) where n is JSON file size
- **Parsing**: O(n) single-pass parser
- **Nested Access**: O(d) where d is nesting depth
- **Memory**: Entire config in memory (typical game configs <1MB)
- **Hot-Reload Check**: O(1) filesystem stat call

## Design Decisions

1. **Built-in JSON Parser**: Eliminated nlohmann/json dependency for cleaner integration
2. **Variant-based Storage**: Used std::variant for type-safe value storage
3. **Template API**: Compile-time type checking with minimal runtime overhead
4. **Shared Pointers**: For arrays/objects to enable efficient copying
5. **Dot Notation**: String parsing for intuitive nested access
6. **Optional Returns**: Explicit error handling without exceptions

## Integration with Engine

The config system integrates seamlessly with existing engine components:

```cpp
class GameEngine {
    std::unique_ptr<Config> config_;

    void initialize() {
        config_->loadFromFile("engine.json");

        // Initialize subsystems with config
        renderer_->initialize(
            config_->get<int>("graphics.width", 1920),
            config_->get<int>("graphics.height", 1080)
        );

        audioEngine_->setVolume(
            config_->get<float>("audio.masterVolume", 0.8f)
        );
    }
};
```

## Thread Safety

⚠️ **Not thread-safe by default**. Use external synchronization (std::shared_mutex) for multi-threaded access.

## Limitations

- No schema validation (values not range-checked)
- No comment support in JSON (use separate documentation)
- No cyclic reference support
- Entire config loaded into memory
- ASCII-only for simplicity (unicode via escape sequences)
